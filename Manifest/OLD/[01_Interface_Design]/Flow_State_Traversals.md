# Flow State Traversals

## Concept: Seamless Data Injection
*De-Phased* menus are not static overlays; they are active engine scenes.

## Principles of Interaction
- **Background Registry**: The menu backdrop utilizes the live engine physics environment, visualizing acoustic pulse data to create an atmospheric, responsive background.
- **Operating Room (Data Injection)**: The character setup process is an in-engine scene. When a player adjusts attributes, they are performing direct data injection into `GContext` (Entity 0). 
- **The Wrapper**: The menu is simply a UI wrapper for the underlying registry. By the time a player initiates "Chapter 1," their chosen race, difficulty, and traits are already committed to the engine’s permanent memory grid.
